# Agent 改进方案 - 基于纯图片推理

## 🎯 核心理念

**重要澄清**:
1. **用户描述是后验信息**：用于验证模型输出，不作为输入
2. **模型只能看图片**：必须完全依靠视觉推理
3. **位置信息限制**：只提取图片中能看到的（门牌号、标识牌），看不到的不要猜
4. **简化输出字段**：只保留核心维修信息，不要元数据

---

## 📊 数据集分析结果

基于 52 条真实报修案例的分析：

### 核心问题

1. **图片信息不足时的推理能力**
   - 27% 的案例图片信息有限
   - Agent 需要从有限的视觉线索推断完整需求

2. **物件识别精确度**
   - 需要识别 11+ 种办公设施
   - 当前识别过于宽泛（"空调"而非"空调出风口盖板"）

3. **故障分类不够精确**
   - 无法区分"物理损坏"（脱落）和"功能故障"（不亮）
   - 19% 物理损坏，13% 功能故障，需要明确区分

4. **缺少紧急程度判断**
   - 漏水是紧急的，挂画框不紧急
   - 需要根据影响范围和安全性评估

---

## 🔧 改进方案

### 方案 1: 优化 System Prompt（核心改进）

**文件**: `app/core/prompts_v2.py`

#### 关键改进点

**1. 强调纯图片推理**
```python
⚠️ 重要：你只能看到图片，没有任何文字描述或背景信息。
你必须完全依靠视觉推理。
```

**2. 细化物件识别指导**
```python
✅ 具体描述示例：
- "空调出风口盖板"（而非"空调"）
- "门禁刷卡器"（而非"门"）
- "天花板筒灯"（而非"灯"）
```

**3. 提供视觉推理技巧**
```python
【视觉推理技巧】
1. 观察物件状态：缺口、裂痕、脱落、水渍、松动
2. 观察环境线索：用户手持物件、物件分离、工具在旁
3. 推断用户意图：
   - 物件分离 + 固定痕迹 → 脱落
   - 手持物件 + 面对墙面 → 需要安装
   - 设备完好 + 无损坏 → 功能故障
```

**4. 优化故障分类**
```python
"物理损坏"      # 脱落、松动、裂开、破损
"功能故障"      # 不亮、黑屏、无反应
"异常状态"      # 漏水、闪烁、显示异常
"安装/加固需求"  # 需要安装或加固
"未知/需人工确认"
```

**5. 限制位置信息提取**
```python
【位置信息提取】
⚠️ 重要：只提取图片中能直接看到的位置信息！

可以提取的：
- ✅ 门牌号（如"会议室 A"）
- ✅ 标识牌（如"茶水间"、"卫生间"）
- ✅ 明显的区域标识（如"健身区"）

不要提取的：
- ❌ 楼层号（图片看不到）
- ❌ 具体房间号（图片看不到）
- ❌ 建筑物名称（图片看不到）

如果图片中没有明显的位置标识，location 字段留空（null）。
```

**6. 添加紧急程度判断**
```python
- High (紧急): 漏水、电路故障、安全隐患、影响多人
- Medium (一般): 单个设备故障、物理损坏但不影响安全
- Low (不紧急): 美观问题、小修小补、可延后处理
```

### 方案 2: 简化 JSON Schema

**文件**: `app/models/schemas_v2.py`

#### 输出字段（8个核心字段）

```python
{
    "is_valid_image": bool,           # 图片是否有效
    "rejection_reason": str | null,   # 驳回原因
    "object_name": str | null,        # 物件名称（具体）
    "issue_description": str | null,  # 问题描述
    "reasoning": str | null,          # 推理过程
    "category": str,                  # 问题分类
    "confidence": str,                # 置信度
    "location": str | null,           # 位置（仅图片可见）
    "urgency": str                    # 紧急程度
}
```

#### 移除的字段
- ❌ `processing_time` - 不需要
- ❌ `model` - 不需要
- ❌ `image_size` - 不需要
- ❌ `subcategory` - 简化，不需要

## ✅ 总结

### 核心改进
1. **强化纯图片推理**：明确告知模型只能看图片
2. **细化物件识别**：提供具体示例和分类清单
3. **优化故障分类**：区分物理损坏和功能故障
4. **添加紧急程度**：根据影响范围和安全性评估
5. **限制位置提取**：只提取图片可见的位置信息
6. **简化输出字段**：移除不必要的元数据
