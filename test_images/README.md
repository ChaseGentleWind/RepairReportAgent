# 测试图片说明

## 目录说明

本目录用于存放测试图片，用于测试报修图片智能分析功能。

## 测试图片要求

### 有效图片示例

**硬件损坏/老化类**:
- 空调出风口盖板脱落
- 门锁损坏
- 水龙头漏水
- 灯具破损
- 墙面开裂

**安装/加固需求类**:
- 需要安装的设备
- 松动的固定件
- 需要加固的结构

**异常状态/环境问题类**:
- 漏水痕迹
- 异味来源
- 温度异常
- 噪音源

**调试/设置需求类**:
- 需要调试的设备
- 需要设置的系统
- 配置问题

### 无效图片示例

**会被驳回的图片**:
- 过暗的图片（光线不足）
- 模糊的图片（失焦）
- 无关内容（风景、人物等）
- 空白图片
- 纯文字图片

## 拍摄建议

### ✅ 好的拍摄方式

1. **光线充足**: 确保拍摄环境光线充足
2. **聚焦清晰**: 对准故障物件，确保清晰
3. **距离适中**: 不要太远或太近
4. **角度正面**: 正面拍摄，避免侧面或仰视
5. **突出重点**: 聚焦于故障部位，减少无关背景

### ❌ 避免的拍摄方式

1. **逆光拍摄**: 导致图片过暗
2. **抖动模糊**: 手持不稳导致模糊
3. **距离过远**: 无法看清细节
4. **角度不佳**: 侧面或仰视角度
5. **背景杂乱**: 过多无关物件

## 测试流程

### 1. 准备测试图片

将测试图片放入本目录：

```
test_images/
├── broken_ac.jpg           # 空调损坏
├── door_lock_broken.jpg    # 门锁损坏
├── water_leak.jpg          # 漏水
├── light_broken.jpg        # 灯具损坏
└── invalid_dark.jpg        # 无效图片（过暗）
```

### 2. 使用命令行测试

```bash
# 测试单张图片
python scripts/test_client.py test_images/broken_ac.jpg

# 批量测试
for img in test_images/*.jpg; do
    echo "测试: $img"
    python scripts/test_client.py "$img"
    echo "---"
done
```

### 3. 使用前端界面测试

1. 启动后端服务: `python main.py`
2. 启动前端服务: `cd frontend && python -m http.server 5500`
3. 访问: http://localhost:5500
4. 上传测试图片查看结果

## 获取测试图片

### 方法一: 自己拍摄

使用手机或相机拍摄实际的报修场景。

### 方法二: 使用示例图片

可以从以下来源获取测试图片：

1. **网络搜索**: 搜索"设备损坏"、"设施故障"等关键词
2. **图库网站**: Unsplash, Pexels 等免费图库
3. **模拟场景**: 拍摄家中的设施（如门锁、灯具等）

### 方法三: 生成测试图片

使用 AI 图像生成工具生成测试图片（如 DALL-E, Midjourney）。

## 注意事项

1. **版权**: 确保使用的图片有合法使用权
2. **隐私**: 不要使用包含个人隐私信息的图片
3. **真实性**: 尽量使用真实的报修场景图片
4. **多样性**: 准备不同类型、不同场景的测试图片
5. **质量**: 包含高质量和低质量的图片用于测试
