# 前端界面使用指南

## 功能介绍

这是一个简单的 Web 前端界面，用于测试报修图片智能分析 API。

### 主要功能

- 📸 **图片上传**: 支持点击上传或拖拽上传
- 🖼️ **图片预览**: 实时预览上传的图片
- 🤖 **AI 分析**: 调用后端 API 进行智能分析
- 📊 **结果展示**: 美观地展示分析结果
- ⚠️ **错误处理**: 友好的错误提示

## 快速开始

### 方法一：使用 Python 内置服务器（推荐）

1. **启动后端服务**

```bash
# 在项目根目录
python main.py
```

2. **启动前端服务**

```bash
# 在项目根目录，打开新终端
cd frontend
python -m http.server 5500
```

3. **访问前端**

打开浏览器访问: http://localhost:5500

### 方法二：使用 VS Code Live Server

1. **安装 Live Server 插件**
   - 在 VS Code 中搜索并安装 "Live Server" 插件

2. **启动后端服务**

```bash
python main.py
```

3. **启动前端**
   - 右键点击 `frontend/index.html`
   - 选择 "Open with Live Server"

4. **访问前端**

浏览器会自动打开: http://127.0.0.1:5500/frontend/index.html

### 方法三：直接打开 HTML 文件

1. **启动后端服务**

```bash
python main.py
```

2. **打开前端**
   - 直接双击 `frontend/index.html` 文件
   - 或在浏览器中打开该文件

**注意**: 某些浏览器可能会阻止本地文件的跨域请求，推荐使用方法一或方法二。

## 使用步骤

### 1. 上传图片

有两种方式上传图片：

- **点击上传**: 点击上传区域，选择图片文件
- **拖拽上传**: 直接将图片拖拽到上传区域

支持的格式: JPEG, PNG, WebP, GIF, BMP, TIFF

### 2. 预览图片

上传后会自动显示图片预览，可以点击右上角的 ✕ 按钮清除图片。

### 3. 开始分析

点击 "开始分析" 按钮，系统会调用后端 API 进行智能分析。

### 4. 查看结果

分析完成后会显示：

**有效图片**:
- ✅ 识别物件
- 📝 故障描述
- 🧠 推理原因
- 📂 问题分类
- 🎯 置信度（High/Medium/Low）
- ⏱️ 处理时间、模型信息、图片尺寸

**无效图片**:
- ❌ 驳回原因
- 💡 改进建议

## 界面说明

### 上传区域
- 蓝色虚线框，支持点击和拖拽
- 鼠标悬停时会有动画效果

### 图片预览
- 显示上传的图片
- 右上角有清除按钮

- 绿色边框表示有效图片
- 红色边框表示无效图片

  - 🟢 High: 绿色（高置信度）
  - 🟡 Medium: 黄色（中等置信度）

### 错误提示
- 红色背景显示错误信息
- 提供重试按钮

## 常见问题

### Q: 提示"无法连接到服务器"

**A**: 请确保后端服务已启动

```bash
python main.py
```

检查服务是否运行在 http://localhost:8000

### Q: 图片上传后没有反应

**A**: 检查以下几点：
1. 图片格式是否支持（JPEG, PNG, WebP, GIF, BMP, TIFF）
2. 图片大小是否超过 10MB
3. 浏览器控制台是否有错误信息（按 F12 查看）

### Q: 分析失败

**A**: 可能的原因：
1. 后端服务未启动
2. API Key 未配置或无效
3. 网络连接问题
4. 图片格式不支持

检查后端日志获取详细错误信息。

### Q: 跨域错误（CORS Error）

**A**: 确保：
1. 使用 Python HTTP 服务器或 Live Server 启动前端
2. 后端 `.env` 文件中的 CORS 配置正确
3. 不要直接双击打开 HTML 文件（某些浏览器会阻止）

## API 配置

前端默认连接到本地后端服务：

```javascript
const API_BASE_URL = 'http://localhost:8000';
const API_ENDPOINT = `${API_BASE_URL}/api/v1/analyze-repair`;
```

如需修改，请编辑 `script.js` 文件中的 `API_BASE_URL` 变量。

## 性能优化建议

1. **图片压缩**: 上传前可以先压缩图片，减少传输时间
2. **网络环境**: 确保网络连接稳定
3. **图片质量**: 使用清晰、光线充足的图片可以提高识别准确率
