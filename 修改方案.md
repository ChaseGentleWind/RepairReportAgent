二、 工作流架构设计 (Pipeline Design)

请将原有的单次调用拆分为串行的两步。虽然多了一次调用，但第二步是纯文本模型，耗时通常在 0.5 秒以内，整体实时性依然能得到保障。

节点 1：视觉感知节点 (Visual Perception Node)

模型：多模态大模型 (如 qwen-vl-plus 或 qwen3.5-omni)

任务：只做“眼睛”，客观描述图片内容，绝对不要去猜测用户的报修意图，也不用输出复杂的 JSON。

输入：图片 + 简单的文本 Prompt。

输出：一段纯文本的客观描述。

节点 2：业务推理节点 (Business Logic Node)

模型：纯文本大模型 (如 qwen-max 或 gpt-4o-mini) —— 纯文本模型在逻辑推理和 JSON 格式控制上远强于多模态模型！

任务：做“大脑”，根据节点 1 传过来的客观描述，套用业务规则（分支 A/B/C/D），推导出最终结论，并严格输出 JSON。

输入：节点 1 的文本描述。

输出：严格的 RepairIntentResponse JSON。
